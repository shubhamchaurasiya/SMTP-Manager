Frequently asked questions

Is SMTP Fallback free to use?

Yes, the free version supports 1 WordPress site with basic SMTP configuration and email fallback protection. You can use it forever at no cost. The Pro plan unlocks unlimited sites, the central dashboard, push updates, and real-time monitoring.

Which SMTP providers are supported?

Gmail, Outlook, SendGrid, Mailgun, Amazon SES, and any custom SMTP server. If it speaks standard SMTP, SMTP Fallback works with it. You can configure up to 2 providers — a primary and a backup — and we handle the switching automatically.

Will it work with Cloudflare-protected sites?

Yes! We use the WordPress REST API which Cloudflare always allows through. Unlike other plugins that rely on direct PHP file access (which can be blocked), SMTP Fallback operates entirely through the standard WordPress REST API at /wp-json/. No firewall rules needed.

How does the automatic fallback work?

When your primary SMTP fails (connection error, auth failure, or timeout), we instantly retry via your configured backup provider. The switch happens transparently inside WordPress's wp_mail() function — no changes needed in your themes or plugins. The entire failover takes under 2 seconds.

Do I need technical knowledge to set it up?

No. The plugin auto-registers with the dashboard on activation and has a guided setup flow. Most users are fully up and running in under 5 minutes. If you can install a WordPress plugin and enter an email address, you can use SMTP Fallback.
